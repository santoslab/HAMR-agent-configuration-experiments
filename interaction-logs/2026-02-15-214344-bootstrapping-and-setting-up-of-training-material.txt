⏺ Here's an overview of this project:

  Goal

  This project is about creating training/context material for Claude so that it
   can effectively assist software engineers working with the HAMR toolchain --
  a model-based development framework for building high-assurance safety- and
  security-critical systems (part of the Sireum framework from Kansas State
  University).

  The idea is that when a developer starts a new HAMR project, they can point
  Claude at the training material in this repo to give it the domain knowledge
  it needs to help with HAMR development tasks.

  Structure

  The project has two main folders:

  hamr-claude-training/ -- The training material for Claude

  - CLAUDE.md -- Top-level guide telling Claude what's in this folder and how to
   use it
  - doc/ -- Documentation about HAMR:
    - hamr-overview.md -- Comprehensive overview of HAMR's purpose, workflow
  (modeling in SysMLv2, code generation, component implementation in Rust,
  testing with PropTest, verification with Verus, deployment to seL4 Microkit)
    - technical-approach.md -- Execution semantics (read-compute-write model,
  port freezing, periodic/sporadic dispatch)
    - folder-structure.md -- Placeholder (TBD)
  - examples/ -- Two worked HAMR examples (Producer-Consumer variants) showing
  SysMLv2 models, generated Rust code, and developer-written application logic
  - .mcp.json -- Configures the Sireum MCP server so Claude can invoke HAMR
  tools (type checking, code generation, etc.) directly

  evaluation-project/ -- A test project to evaluate the training material

  - Contains a Simple Network Guard (SNG) specification -- a toy network
  guard/cross-domain solution with gate and filter pipeline stages
  - Currently has requirements but the sysmlv2/ and hamr/ folders appear empty
  -- this is the project where you'd ask Claude to apply HAMR using the training
   material, then evaluate how well it performs

  Methodology

  The workflow is iterative:
  1. Ask Claude to consult hamr-claude-training/ for context
  2. Ask Claude to perform HAMR development tasks on the evaluation-project/
  3. When Claude struggles or makes mistakes, improve the training material
  4. Repeat

  Current State

  The training material has solid documentation and examples, but some pieces
  are still in progress (e.g., folder-structure.md is TBD). The evaluation
  project has requirements written but implementation hasn't been started yet.
  The project also has Sireum MCP integration configured at both the root and
  training-folder level, giving Claude access to HAMR's CLI tools (type
  checking, code generation, etc.).
